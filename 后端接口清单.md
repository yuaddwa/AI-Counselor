# AI辅导员小程序 - 后端接口清单

> 基于前端 `common/utils/request.js` 中已定义的接口 + 各页面数据模型推导
>
> 统一规范：所有接口返回 JSON，登录态通过 `Authorization: Bearer <token>` 传递，401 自动跳转登录页

---

## 1. 统一响应格式

```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录/token过期 |
| 403 | 无权限 |
| 500 | 服务器错误 |

## 2. 登录模块

### 2.1 登录

| 项目 | 内容 |
|------|------|
| URL | `POST /api/login` |
| 说明 | 学生/辅导员共用同一登录接口，根据角色返回不同信息 |

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 学号或工号 |
| password | string | 是 | 密码 |
| role | string | 是 | `student` 或 `counselor` |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | JWT令牌 |
| userInfo | object | 用户信息 |

**userInfo 对象（学生）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 学号，如 `"2026001"` |
| name | string | 姓名，如 `"张同学"` |
| className | string | 班级，如 `"计算机2601"` |
| role | string | 固定 `"学生"` |
| avatar | string | 头像URL（可选） |

**userInfo 对象（辅导员）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 工号，如 `"T001"` |
| name | string | 姓名，如 `"李老师"` |
| role | string | 职务，如 `"辅导员"` / `"高级管理员"` |
| avatar | string | 头像URL（可选） |

## 3. 学生端 - 首页/对话模块

### 3.1 获取首页数据

| 项目 | 内容 |
|------|------|
| URL | `GET /api/student/home` |
| 说明 | 首次进入首页加载的数据 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| quickTags | string[] | 快捷标签列表，如 `["新生报到","宿舍后勤","缴费","奖助","请假","军训"]`（可选，前端有默认值） |
| recentSessionId | string | 最近会话ID，用于续接对话（可选） |

### 3.2 AI对话（核心接口，对接LangChain）

| 项目 | 内容 |
|------|------|
| URL | `POST /api/chat/send` |
| 说明 | 发送消息给AI，后端调用LangChain返回回答 |

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户输入的问题文本 |
| sessionId | string | 否 | 会话ID，新对话不传，续接对话传入 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| reply | string | AI回答文本 |
| source | string | 来源文档名称，如 `"新生入学指南"` |
| sessionId | string | 本次会话ID |

### 3.3 获取历史对话列表

| 项目 | 内容 |
|------|------|
| URL | `GET /api/chat/history` |
| 说明 | 学生的历史对话会话列表 |

**请求参数（query）**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | number | 否 | 页码，默认1 |
| pageSize | number | 否 | 每页条数，默认20 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| total | number | 总条数 |
| list | array | 会话列表 |

**list 单项**

| 字段 | 类型 | 说明 |
|------|------|------|
| sessionId | string | 会话ID |
| title | string | 会话标题/最后消息摘要（前端同时兼容 `lastMessage` 字段） |
| updateTime | string | 最后更新时间，如 `"2026-05-20 14:30"`（前端会做 `T` 替换空格处理） |

### 3.4 获取历史对话详情

| 项目 | 内容 |
|------|------|
| URL | `GET /api/chat/session/:id` |
| 说明 | 获取某个会话的完整消息记录 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| sessionId | string | 会话ID |
| messages | array | 消息列表 |

**messages 单项**

| 字段 | 类型 | 说明 |
|------|------|------|
| role | string | `user` 或 `assistant`（前端会将 `assistant` 映射为 `ai`） |
| content | string | 消息文本 |
| timestamp | string | 发送时间戳（ISO格式或时间戳均可） |

### 3.5 删除历史对话

| 项目 | 内容 |
|------|------|
| URL | `DELETE /api/chat/session/:id` |
| 说明 | 删除单条历史对话 |

**响应 data** 无

### 3.6 清空所有对话

| 项目 | 内容 |
|------|------|
| URL | `DELETE /api/chat/clear` |
| 说明 | 清空当前用户所有历史对话 |

**响应 data** 无

### 3.7 提交反馈（对话内点赞/点踩）

| 项目 | 内容 |
|------|------|
| URL | `POST /api/feedback` |
| 说明 | 对AI某条回答提交正/负反馈 |

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sessionId | string | 是 | 会话ID |
| messageIndex | number | 是 | 消息在会话中的序号 |
| type | string | 是 | `like` 或 `dislike` |

**响应 data** 无

## 附录C：AI对话与LangChain对接说明

`POST /api/chat/send` 是核心接口，后端收到学生消息后需要：

1. 根据 `sessionId` 获取历史上下文
2. 将用户消息 + 历史上下文发送给 LangChain 服务
3. LangChain 从知识库检索匹配内容，生成回答
4. 后端将回答返回前端，同时附带命中的知识库来源

**LangChain 服务需要提供的能力：**
- 接收 `{ message, history[] }` 输入
- 返回 `{ reply, source }` 输出
- 支持知识库向量检索
- 支持多轮对话上下文

**请假意图识别：** 如果 LangChain 检测到用户意图是请假（或前端直接标记），后端可跳过 LangChain 直接返回引导流程标记，前端进入请假收集流程。
